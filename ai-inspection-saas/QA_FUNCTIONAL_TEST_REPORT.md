## 1. 测试结论

当前版本已完成 MVP 主链路中的部分能力：登录、多项目成员隔离、摄像头列表、点位读取与保存校验、地图上传安全校验、媒体播放会话、告警状态机、规则列表和审计日志展示。

本轮实际执行 29 个接口/页面验证点，22 个通过，7 个失败或缺失。阻塞 MVP 完整验收的核心问题集中在：

- 角色级权限未落地：中控值班员可以查看审计日志、保存点位配置。
- 多项目切换链路不完整：admin 切换到空项目 `mall_002` 后，大屏选择器变更但页面仍展示 `mall_001` 旧数据。
- 平台规则配置只有列表，没有新增、编辑、启停、复制、删除。
- 第三方 AI 事件接入接口缺失，无法验证“AI 事件 -> 规则过滤 -> 告警生成 -> 大屏推送”主链路。
- 实时订阅 SSE/WebSocket/token 缺失，当前大屏不是实时闭环。
- 证据下载令牌、删除、附件、导出等证据审计链路缺失。

页面验证结果：

- 登录后默认进入指挥中心，能展示大屏指标、实时告警列表、L1 地图点位、视频轮巡。
- 管理入口可进入摄像头管理，摄像头列表展示状态、媒体状态、AI 能力和预览/定位入口。
- 摄像头预览会创建短时播放会话并弹出提示。
- admin 切换到 `mall_002` 后，下拉框选中 `虹桥商业裙楼`，但标题、指标、告警、地图、视频轮巡仍为 `大悦城示例项目`，存在项目串数风险。

## 5. 回归建议

1. 先补齐角色权限中间件，所有写操作、审计、规则、证据、导出接口必须按 `tenantId + projectId + role + dataScope` 拦截。
2. 修复项目切换：先清空项目级 UI 状态，再请求新项目 bootstrap 数据；空项目返回空态而不是 404 导致旧页面残留。
3. 补齐 AI Event Adapter 路由和规则引擎，至少覆盖签名、nonce、幂等、映射失败、低置信度丢弃、冷却去重、生成告警。
4. 补齐实时订阅 token 与 SSE/WebSocket，项目切换时关闭旧订阅。
5. 补齐规则 CRUD、证据下载/删除/附件/导出接口与审计。
6. 将本报告中的 P0/P1 用例固化为自动化回归，尤其是权限、项目隔离、点位版本、告警状态机和 AI 事件幂等。
